A Journey Thru Frontend Accessibility

This website is self-practice of building accessible web components under the direction of the blog Inclusive Components.

The components I will practice building are:

  • Cards

    The multivarious card component, done inclusively

  • Data Tables

    Create accessible displays of data

  • Notifications

    Alert users to information in an accessible manner

  • Content Sliders

    Slide through content with inclusivity in mind

  • Collapsible Sections

    Provide sections that can be closed and opened accessibly

  • Tabbed Interfaces

    Tab through content in an accessible manner

  • Theme Switchers

    Switch those color themes inclusively!

  • Tooltips & Toggletips

    Clarifying the use of tools to users

  • Menus & Menu Buttons

    What is a menu and how to make it accessible?

  • Todo Lists

    The classic todo list example, done accessibly

  • Toggle Buttons

    Creating an accessible toggle button

This navigation was the card component practice! I learned that cards should be in a list element, that links should only wrap a section of the card e.g. the heading, that styles should be controlled so that different text lengths and image sizes shouldn't break the card, to not add too much functionality, and that headings should always begin sections.
